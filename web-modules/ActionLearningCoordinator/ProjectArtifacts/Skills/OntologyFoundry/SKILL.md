# Ontology Foundry Skill

Private action and entity roots for the Ontology Foundry agent.

EvidenceGraph v2 exposes narrow wrappers for persisted historical entity-search episode reconstruction, current ontology inspection, historical conversation navigation, and one fixed working artifact. It reconstructs exact model-generated probes, ranked historical candidates, pairing diagnostics, and exact downstream prototype use before current replay and context verification.

The superseded session-key message-row `SessionEntitySearchEpisodeDriver` wrapper is not part of the active skill. Historical query-fragment navigation remains tabled until the corrected fragment-first owner is available; ordinary Expand-and-Enhance uses bounded session-message evidence and current ontology inspection only.

Mode B Expand adds role-specific session-message search and direct-descendant inspection. It writes six evidence-grounded expansion sections into the same EvidenceGraph document and never promotes aliases or writes ontology objects.

Session indexing has one canonical flow: GatherAndRank asks one Level2 for a ranked evidence-grounded entity list; ResolveRanked binds exact existing prototypes and converts them into a proposed session-owned `RelatedEntities` map whose Level2 Importance values are local `RelativeValue` values. NewCandidate and Ambiguous labels remain unresolved index evidence and do not trigger ontology construction. Harvest is the bounded-cohort orchestrator for those same read-only stages and writes the same `Ranked Level2 entities`, `Ranked entity resolutions`, and `Proposed Session Entity Index` sections. MaterializeSessionEntityIndex is the separate explicitly authorized mutating stage: it re-verifies confirmed mappings and persists one stable `Level1Session` definition through NL Memory in the indexed session. ProposeBuild remains a separate explicitly promoted construction workflow. Expand and Enhance are one plan-first operation: it discovers evidence, resolves concepts, compares current schema, classifies every finding, and emits exact typed proposals only for enhancement-ready rows. Generalize starts from one grounded physical or operational seed, follows its real enclosing relationship, selects a typed authoritative enumerator, discovers and resolves peers, and only then determines organizing parents, missing objects, relationships, and shared structure. Global ontology apply modes remain intentionally blocked until a scoped runtime approval/backup/compile/restore coordinator is installed; raw ontology authoring tools are not exposed to the child.
