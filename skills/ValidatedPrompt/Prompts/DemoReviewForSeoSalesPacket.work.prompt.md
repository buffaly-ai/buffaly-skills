# Demo Review For SEO Sales Packet — Work Prompt

Review the demo website artifact before proposal deck or packet use.

Inputs: original website URL, validated analysis or client report, demo website artifact or HTML/content, screenshots if available. If screenshots are absent, review supplied HTML/content and mark screenshot checks `Not verified`.

Required work:
1. Check whether the demo addresses original website weaknesses.
2. Check whether the business is easier to understand.
3. Check whether CTAs/contact path are stronger.
4. Check whether local positioning and service coverage are clearer.
5. Check whether anything appears visually awkward, unsupported, misleading, or not evidence-safe.
6. Identify required fixes before proposal/deck use.
7. Decide whether the demo is ready, needs revision, or blocked.

Output headings:
1. Demo review decision: `ready`, `needs revision`, or `blocked`
2. Original weaknesses addressed
3. Visual/layout issues
4. Content and evidence issues
5. CTA/local positioning review
6. Sales-usefulness notes
7. Required fixes before deck
8. Not verified items
