# Client-Facing Website Improvement Report — Validation Criteria

Validate whether the candidate result is a client-ready website improvement report based on supplied analysis and positioning.

Pass only if all are true:
1. It references the target website or business.
2. It is client-facing and does not expose internal validator/process instructions.
3. It includes an executive summary and action plan.
4. It covers SEO/local SEO, conversion path, trust/proof, content/service pages, competitor positioning, and AI-search/LLM readiness.
5. Recommendations are tied to supplied evidence or marked as needing verification.
6. It avoids unsupported claims about ratings, rankings, licenses, ownership, years in business, or response times.
7. It explains business impact in plain language.
8. It includes quick wins and longer-term recommendations.
9. It includes a demo/rebuild opportunity summary.
10. It is specific enough for a sales packet.

Fail if it lacks major sections, invents facts, sounds internal, or is not grounded in upstream analysis.
