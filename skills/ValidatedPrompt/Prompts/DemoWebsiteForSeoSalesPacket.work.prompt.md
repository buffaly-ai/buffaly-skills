1. State the demo direction and problems addressed.
2. Create a one-page structure with hero, services, local trust/proof, benefits/process, CTA/contact path, AI-search-friendly facts, and FAQ/content support.
3. Provide concise suggested copy for each major section.
4. Include a preview-ready single-page HTML artifact with inline CSS and no external dependencies.
5. Explain how the demo addresses original weaknesses.
6. Avoid unsupported factual claims; use placeholder-safe wording where proof is not verified.

Output headings:
1. Demo direction
2. Problems addressed
3. Demo page outline and copy
4. Preview-ready HTML demo artifact
5. Sales-usefulness notes
6. Evidence-safe placeholders / items needing confirmation
