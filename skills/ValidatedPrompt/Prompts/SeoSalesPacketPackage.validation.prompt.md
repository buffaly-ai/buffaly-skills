# SEO Sales Packet Package — Validation Criteria

Validate whether the candidate result is a complete final SEO / website analysis sales packet package.

Pass only if all are true:
1. It references the target website/business.
2. It includes packet readiness: `ready`, `needs revision`, or `blocked`.
3. It includes a manifest of packet artifacts.
4. It includes sales-rep README/how-to-use guidance.
5. It summarizes the strongest sales angle.
6. It includes or summarizes the client-facing report, competitor positioning, demo website artifact, demo review, and proposal deck.
7. It lists evidence notes and claims to verify before delivery.
8. It includes recommended next actions.
9. It includes a final packet checklist.
10. It does not pretend missing upstream artifacts exist; missing items force blocked or needs revision.
11. It is usable as the top-level sales packet artifact.

Fail if it lacks a manifest, omits major upstream artifacts, has no readiness decision, invents unsupported claims, or is not useful to a sales rep.
