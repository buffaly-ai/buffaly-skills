# Website Analysis From Evidence For SEO Sales Packet — Work Prompt

Create the website analysis section for an SEO / website improvement sales packet using only the supplied evidence dossier.

Do not browse, search, fetch pages, or call tools. Treat the supplied dossier as the inspected evidence. If a required item is missing, mark it `Not verified` and explain what would be needed. Do not invent facts.

Required work:
1. Identify the target website, business, industry, and visible market from the dossier.
2. Summarize homepage metadata, crawlability, page inventory, local SEO/NAP, service coverage, content quality, conversion path, trust/proof, reviews/social proof, accessibility/media, AI-search/LLM readiness, competitor positioning, performance/template risk, and prioritization.
3. Tie every finding to supplied evidence or mark it `Not verified`.
4. Produce a compact but complete client/sales-useful audit.
5. Include competitor analysis using only supplied competitor evidence.

Output headings:
1. Executive summary
2. Quick scorecard
3. What is working well
4. Main issues and opportunities
5. LLM information quality / AI-search readiness
6. Competitor analysis
7. High-priority action plan
8. Quick wins
9. Longer-term SEO/content recommendations
10. Demo/rebuild opportunity summary
11. Lead-quality or sales notes
12. Acceptance matrix

The acceptance matrix must include rows for: homepage metadata, local SEO and NAP, technical crawlability, page inventory, service keyword coverage, content quality/differentiation, conversion path, trust/E-E-A-T, reviews/social proof, accessibility/media, AI-search/LLM readiness, competitor landscape, competitor page gaps, performance/template risk, and quick wins/prioritization.
