# SEO Sales Packet Package — Work Prompt

Assemble the final SEO / website analysis sales packet from validated upstream artifacts.

Inputs: website URL, validated website analysis, required SEO review notes, competitor positioning, client-facing report, demo website artifact, demo review, and proposal deck content. If a required artifact is missing, mark the package `blocked` and list what is missing.

Required work:
1. Create a packet manifest naming each included artifact.
2. Create a README / landing page for the sales rep.
3. Summarize the strongest sales angle.
4. Summarize how to use the demo and proposal deck.
5. List evidence notes and claims to verify before prospect delivery.
6. Provide recommended next actions.
7. Clearly state whether the packet is ready, needs revision, or blocked.

Output headings:
1. Packet readiness: `ready`, `needs revision`, or `blocked`
2. Sales packet manifest
3. Sales rep README / how to use this packet
4. Strongest sales angle
5. Included client-facing report summary
6. Demo website summary and usage notes
7. Proposal deck summary and presentation path
8. Evidence notes / claims to verify
9. Recommended next actions
10. Final packet checklist
