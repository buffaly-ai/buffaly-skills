# Demo Website For SEO Sales Packet — Validation Criteria

Validate whether the candidate result is a usable demo website artifact.

Pass only if all are true:
1. It references the target website/business or supplied analysis.
2. It states the demo direction.
3. It explains which original weaknesses the demo addresses.
4. It includes a concrete page outline and section copy.
5. It includes a preview-ready HTML artifact or clearly delimited demo page artifact.
6. The demo has hero/message, service coverage, local trust/proof area, CTA/contact path, and content useful for visitors and AI assistants.
7. It avoids unsupported factual claims or marks them as placeholders/items to confirm.
8. It includes sales-usefulness notes.
9. It is specific enough to include in a packet.
10. It is compact enough to be practical and not mostly boilerplate.

Fail if it omits the demo artifact, does not address validated weaknesses, invents proof claims, or is generic redesign advice.
