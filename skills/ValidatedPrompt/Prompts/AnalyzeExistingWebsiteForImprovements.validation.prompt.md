# Analyze Existing Website For Improvements — Validation Criteria

Validate whether the candidate result is a usable, evidence-backed website improvement audit for the website in the original user input.

Pass only if all of the following are true:

1. The result clearly names or references the target website URL from the original input.
2. The result shows the homepage was inspected and includes concrete observations from it.
3. The result addresses robots.txt and sitemap.xml, or marks each as not verified / unavailable with a reason.
4. The result lists important pages inspected or discovered, or explains why important pages could not be discovered.
5. The result covers local SEO/NAP, crawlability, service keyword coverage, conversion path, trust/social proof, content quality, AI-search/LLM readiness, and competitor positioning at least briefly.
6. The result includes an acceptance matrix or equivalent structured checklist with evidence for the required audit areas.
7. Findings are tied to observed evidence, inspected URLs, or explicitly marked `Not verified`.
8. Competitor analysis separates actual business competitors from directories/lead-generation/SEO-only results when such distinction is relevant; if competitors were not verified, the result says so clearly.
9. The result avoids unsupported claims about ratings, rankings, ownership, years in business, licensing, response times, or other factual claims not shown in evidence.
10. The result includes prioritized quick wins and longer-term recommendations that a small business could understand.
11. The result is client-ready or sales-useful: practical, clear, and not just a list of generic SEO advice.
12. The result does not claim validation passed merely because the prompt asked for validation; it must actually satisfy these criteria.

Fail if the result is mostly generic, skips the matrix, lacks inspected evidence, omits major required categories, invents claims, or says work was done without showing evidence.
