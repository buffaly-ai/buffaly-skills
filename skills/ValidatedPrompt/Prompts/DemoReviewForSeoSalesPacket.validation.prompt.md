# Demo Review For SEO Sales Packet — Validation Criteria

Validate whether the candidate result is a useful required review of a demo website artifact.

Pass only if all are true:
1. It references the original website or supplied demo artifact.
2. It includes a clear decision: `ready`, `needs revision`, or `blocked`.
3. It checks whether the demo addresses original weaknesses.
4. It checks CTA/contact path, local positioning, service clarity, and trust/proof.
5. It checks for unsupported or misleading claims.
6. It includes visual/layout review notes, or marks screenshot-based visual review `Not verified`.
7. It lists required fixes or says no required fixes were found.
8. It includes sales-usefulness notes.
9. It avoids inventing visual evidence that was not supplied.
10. It is specific enough to decide whether to use the demo in a deck.

Fail if it only says the demo looks good, omits the decision, ignores evidence safety, or fails to compare against original weaknesses.
